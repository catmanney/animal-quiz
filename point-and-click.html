Hello beta testers! This is the demo for my animated quiz. On this html page, you should find four animals and a "start quiz" button. Upon pressing that button, a 6-question vocabulary quiz should appear. The correct answer to each question is indicated with a star. This star will not be available in the final product.

Here are some things you might enjoy testing.
-If you answer all 6 questions correctly, each of the four animals should have a unique, extended animation as a reward.
-Under regular circumstances, a correct answer should produce a "wink" in each animal.
-An incorrect answer should produce "bored" facial expression in each animal.
-Pressing the submit button without selecting an option should produce a "please select option" message, but no animation. Doing so does not affect your final quiz score.

Additional things to try
-clicking each animal or their surrounding whitespace should produce a unique animation and message, (some animals like being clicked on, some don't).
-There is a list of buttons above each animal corresponding to the animations I have implemented. This has made it much easier to debug, and hopefully will be useful to you. These buttons will not be available in the final product.

Some problems I am currently working on
-I am happy with the waves for cat and bird, but am a little unsure about the waves for fish and dog.
-I would like each animal to "wave hello" upon the window open (when viewer first sees the page). Sometimes the page finishes loading before the user opens the window and sees the content, so this is a little tricky.
-Similarly, I would like each animal to "wave goodbye", when the page is closed (in coordination with an "are you sure you want to leave?" message prompt).
-I recognize that pressing the "start quiz" button after the test has begun produces additional unnecessary options. However, it seems pretty harmless to the functions. I will be working on this, as there are few conceptual issues I need to still work on. For our purposes, you may simply ignore it (unless you're a rebel).
-The alphabet and font work great on my computer, but there are some issues when viewing on my iPhone. The most important is that when the owl is clicked and the "angry" animation starts, the "V" character is not displayed properly. This character is meant to respresent angry eyebrows, and does so quite well on my computer. However, it is very obviously an alphabetical character when viewed on my iPhone. I have spent a great deal of time trying to fix this without much progress.
-I think some minor changes could be made to the bubbles in the "fish win" event.
-I feel like I use " > < " as eyes a little too much?

That's about it for notes. I love both positive and (respectfully expressed) negative feedback. Feel free to comment on aethetics, timing and format related observations. I would love to know how it displays on phone, computer, or any other device you have!
Thank you for your time, I hope you are entertained!
-Cat
